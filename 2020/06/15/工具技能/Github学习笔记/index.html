工作流程

  • 克隆 Git 资源作为工作目录。
  • 在克隆的资源上添加或修改文件。
  • 如果其他人修改了，你可以更新资源。
  • 在提交前查看修改。
  • 提交修改。
  • 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

基本框架

  1. 工作区: 当前电脑下的工作目录空间
  2. index区: .git文件夹中,保存所有改动目录文件的索引
  3. HEAD: 本地master，保存最新提交（commit）的版本。
  4. oringin/仓库：远端服务器存储的工作空间
